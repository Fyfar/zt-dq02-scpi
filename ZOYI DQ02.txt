; ============================================================
; ZOYI ZT-DQ02 LCR Meter — TestController device definition
;
; Key device behaviors (firmware-verified):
;  - Instrument CONTROL commands are cycle-only: each call advances to
;    the next value in a fixed sequence. No direct-set path for mode,
;    frequency, speed, range, model, voltage, or bias.
;    Exception: COMParator:NOMinal and COMParator:ERRor accept values.
;  - FETCh? is the ONLY state reader, returning 17 comma-separated
;    fields. Individual query commands (FREQuency?, APERture?, etc.)
;    all return empty strings.
;  - Over-range: fields [0] and [1] contain literal "0L" (not a number).
;    Parse carefully — SYSTem:ERRor? still returns "No error" on OL.
;  - MED and FAST aperture have identical sample rate (~0.36 s).
;  - COMParator:NOMinal units depend on mode:
;      R / Z       → ohms (1:1),  e.g. 1000 = 1 kΩ
;      C / ECAP    → nanofarads,  e.g. 47000 = 47 µF
;      L           → microhenries, e.g. 1000 = 1 mH
;  - COMParator:ERRor is capped at 40 % over SCPI (front panel: 99.9 %).
;  - field[16] (PASS/FAIL) freezes when comparator is off (field[12]=0).
;    Always verify CompEnable before trusting CompResult.
; ============================================================

; --- Primary measurement value (selector = field[3] canonical type) ---
#value Capacitance F SI C
#value Inductance H SI L
#value Resistance Ohm SI R,Z
#value BatVoltage V SI V

#value Secondary - D4

; --- Settings captured from FETCh? fields 9, 10, 11 ---

#value Amplitude mV D0

; --- FETCh? returns 17 comma-separated fields.
; Format codes: f=float, x=skip.
; Positions captured: [0]=primary, [1]=secondary,
;                     [9]=freq, [10]=amplitude, [11]=bias
#askValues :FETCh?

; ============================================================
; COMPARATOR — Pass/Fail testing (page: Comparator)
;
; Workflow:
;  1. Set CompNominal (value in mode-correct units)
;  2. Set CompError (tolerance %)
;  3. Check CompEnable to activate
;  4. Read CompResult: 1=PASS, 0=FAIL
;
; WARNING: field[16] FREEZES when comparator is off.
; Always verify CompEnable is checked before trusting CompResult.
; ============================================================

; Enable/disable comparator — reads field[12], writes COMParator 0/1
; (device also accepts COMParator ON/OFF; SCPI bool 0/1 is equivalent)
#cmdSetup checkbox CompEnable Comparator
:read: :FETCh?
:readmath: trim(getElement(value,12,","))
:write: :COMParator #
:update: CompResult CompParam CompNomDisp CompTolDisp
:tip: Enable comparator. PASS/FAIL in CompResult only valid when this is checked (field[12]=1). Result freezes when off.
Comparator 0 1

; Nominal reference value — direct-set command (not a cycle command)
; Unit depends on active mode (read CompParam for current mode):
;   R / Z mode    → ohms         e.g. 1000 = 1 kΩ
;   C / ECAP mode → nanofarads   e.g. 47000 = 47 µF   (NOT microfarads)
;   L mode        → microhenries  e.g. 1000 = 1 mH
; Bare float ONLY — suffixes like u, m, k raise -120 numeric error
#cmdSetup number CompNominal Comparator
:write: :COMParator:NOMinal #
:update: CompNomDisp
:tip: Nominal value. R/Z: ohms; C/ECAP: nF (47000=47µF); L: µH (1000=1mH). Bare float only — no unit suffix.
val 0 1000000000

; Tolerance percentage — direct-set command (not a cycle command)
; SCPI firmware clamps any value >= 40 to exactly 40.0 %
; (front-panel menu allows up to 99.9 %; SCPI limitation only)
#cmdSetup number CompError Comparator
:read: :FETCh?
:readmath: trim(getElement(value,15,","))
:write: :COMParator:ERRor #
:update: CompTolDisp
:tip: Tolerance ±%. SCPI clamps any value >=40 to 40.0. Front-panel menu allows up to 99.9%.
% 0 40

; PASS/FAIL result from field[16]: 1=PASS, 0=FAIL

:tip: 1=PASS, 0=FAIL. ONLY valid when CompEnable is checked (field[12]=1). Stale value lingers when disabled.
